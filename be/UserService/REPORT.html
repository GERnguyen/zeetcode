User Service Auth Update

This report summarizes the service logic, authentication middleware, and email-confirmation flows added to the UserService.

What Was Added

  • JWT access and refresh token handling with refresh token rotation and revocation.
  • Role and ownership guards for APIs: user-only, self-or-admin, and admin-only checks.
  • Email confirmation links for signup and sensitive changes, plus password reset by email link.
  • Controllers wired to the service functions so routes can be added next.

Key Files

  • src/services/user.service.ts: core auth logic
  • src/middlewares/auth.middleware.ts: access/refresh auth
  • src/controllers/user.controller.ts: controllers
  • src/services/email.service.ts: nodemailer Gmail sender
  • prisma/schema.prisma: refresh/email token models
  • src/config/index.ts: new auth + mail config

Authentication Flow

  • Access token is a short-lived JWT signed with JWT_ACCESS_SECRET.
  • Refresh token is a long-lived JWT signed with JWT_REFRESH_SECRET and stored as a hash in the RefreshToken table.
  • Refresh token rotation revokes the old token and issues a new pair.
  • Middleware functions: authenticateAccessToken, authenticateRefreshToken, requireAdmin, and requireSelfOrAdmin.

Email Confirmation Flow

  • Email links use a one-time token stored in EmailToken with an expiration time.
  • Signup: sends confirmation link.
  • Change password: sends confirmation link with a hashed new password stored in the token payload.
  • Password reset: sends reset link; the token is consumed when the user sets a new password.

New Environment Variables

  • JWT_ACCESS_SECRET
  • JWT_REFRESH_SECRET
  • JWT_ACCESS_EXPIRES_IN (default 15m)
  • JWT_REFRESH_EXPIRES_IN (default 7d)
  • EMAIL_TOKEN_EXPIRES_MINUTES (default 30)
  • MAIL_USER and MAIL_PASS (Gmail app password)
  • MAIL_FROM (optional; defaults to MAIL_USER)
  • FRONTEND_URL (used for link generation)
Required local steps:
  • Run prisma migrate dev and prisma generate.
  • Run npm install to add nodemailer dependencies.
  • Set the new environment variables before running the service.
